LLDPviz Help

      
    

    


      

        
What is LLDPviz?

        
LLDPviz is a network topology visualization tool that builds interactive network diagrams
          from MikroTik LLDP neighbor data. You paste raw CLI output from your routers/switches, and the app
          automatically constructs a visual map of your network.

        
Data is stored locally in your browser (localStorage). Optionally, you can Share a site to generate a link backed by the server.

      


      

        
Getting Started

        
    
          
  1. Create a Site — Enter a name and click Create. A site represents a network/location.
    2. 
          
  2. Collect LLDP data — Log in to a MikroTik device and run:
            /ip/neighbor/print detail where discovered-by~"lldp"
          
    3. 
          
  3. Paste data — Copy the entire output and paste it into the text field. Source Identity is auto-detected from the prompt (e.g. [admin@Router1] >).
    4. 
          
  4. Click "Add Neighbors" — Devices and connections are added to the topology.
    5. 
          
  5. Repeat — Paste output from more devices to build the full network map.
    6. 
        

      


      

        
Traverse Mode

        
Traverse mode is a guided discovery workflow that helps you systematically scan the entire network:

        
    
          
  • Click the Traverse button to start.
    • 
          
  • Paste LLDP output from the current device.
    • 
          
  • New neighbors appear as Pending in the queue.
    • 
          
  • Use the MAC-telnet command (copyable) to jump to the next device.
    • 
          
  • Mark devices as Dead End if they cannot be scanned.
    • 
          
  • The progress bar shows how much of the network has been scanned.
    • 
        

        
While traversing, you can optionally paste additional data per device:

        
    
          
  • Interface Status — /interface/print without-paging
    • 
          
  • IP Addresses — /ip/address/print without-paging
    • 
          
  • Bridge VLANs — /interface/bridge/vlan/print without-paging
    • 
        

        
This extra data is shown in the device detail panel and used by the Ports view.

      


      

        
View Modes

        

          

            Network
            
Force-directed layout. Devices arrange themselves automatically.

          

          

            Tree
            
Hierarchical top-down from a chosen root. Double-click to change root.

          

          

            Radial
            
Concentric circles from root. Good overview of network depth.

          

          

            Text
            
ASCII tree output, copyable for documentation or chat.

          

          

            Table
            
Sortable device list with all fields. Exportable as CSV.

          

          

            Matrix
            
N×N connection grid showing interfaces at each intersection.

          

          

            Ports
            
Switch faceplate view. Shows every port with connection status.

          

        

      


      

        
Toolbar

        
    
          
  • Root selector — Choose the root node for Tree, Radial, and Text views.
    • 
          
  • Edge labels — Toggle interface names on graph connections (Network/Tree/Radial).
    • 
          
  • Search — Filter devices by name, IP, MAC, or board across all views.
    • 
        

      


      

        
Node Shapes & Colors

        

          

            
            Hexagon (orange) — Router + Bridge
          

          

            
            Diamond (red) — Router only
          

          

            
            Box (blue) — Bridge/Switch
          

          

            
            Triangle (green) — WLAN AP
          

          

            
            Dot (grey) — Other/unknown
          

        

      


      

        
Export, Import & Share

        
    
          
  • Export — Downloads the selected site as a .json file.
    • 
          
  • Import — Loads a .json file to create or replace a site.
    • 
          
  • Share — Uploads the site to the server and copies a shareable link. Anyone with the link can view the topology.
    • 
          
  • Save — Saves a new revision of a shared site. Previous revisions remain accessible via the dropdown.
    • 
        

      


      

        
Interaction

        
    
          
  • Click a device to view details (IP, MAC, board, connections, VLANs).
    • 
          
  • Click a connection (edge) to view interface details.
    • 
          
  • Double-click a device (in Tree/Radial mode) to make it the root.
    • 
          
  • Scroll to zoom in/out.
    • 
          
  • Drag to pan around the graph.
    • 
          
  • Drag a device to reposition it manually.
    • 
        

      


      

        
JSON Format Example

        
{
  "name": "Datacenter Nord",
  "devices": {
    "Core-Router": {
      "identity": "Core-Router",
      "macAddress": "48:A9:8A:AD:41:DD",
      "platform": "MikroTik",
      "board": "CCR2004-16G-2S+",
      "version": "7.14",
      "address": "10.0.0.1",
      "systemCaps": "bridge,router",
      "systemCapsEnabled": "bridge,router",
      "scanned": true
    }
  },
  "connections": [
    {
      "fromIdentity": "Core-Router",
      "toIdentity": "SW-Floor1",
      "fromInterface": "sfp-sfpplus1",
      "toInterface": "sfp-sfpplus1",
      "lastSeen": "2026-02-23T12:00:00.000Z"
    }
  ]
}

      


      

        
Format Manifest (for LLM/automation)

        
A JSON Schema manifest is included with the project: lldpviz-format.manifest.json

        
Use the manifest to let an LLM or script generate valid LLDPviz files programmatically — e.g. from an SNMP crawl, Netbox export, or another LLDP source.
